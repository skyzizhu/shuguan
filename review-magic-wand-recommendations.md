# 复审魔法棒推荐文案

## 背景

该魔法棒用于`复审`阶段，不是初审。

触发时机：

- 用户已经完成首轮审核
- 用户认为首轮结果不满意、不准确，或需要更明确的解释
- 用户希望通过多轮对话继续纠偏，并在确认后更新第二屏审核结果

设计原则：

- 不重复初审能力，不再强调“提取文字并审核”“重新跑一遍全量审核”
- 重点围绕“解释依据、定位问题、复核边界、排查误判”
- 文案需要短、清晰、像按钮
- 每类物料控制在 4 个动作以内

## 一，文字类复审魔法棒

推荐按钮：

1. 复核疑似违规内容
2. 定位高风险段落
3. 解释判定依据
4. 排查误判内容

推荐说明：

- 适用于用户直接输入文案内容后的复审
- 重点解决“哪里违规”“为什么违规”“是不是误判”三类问题

## 二，文件类复审魔法棒

推荐按钮：

1. 复核疑似违规内容
2. 定位高风险段落
3. 解释规则与法律依据
4. 排查误判或漏判

推荐说明：

- 适用于 PDF、Word 等文稿文件复审
- 与文字类相比，更强调“段落与原句定位”“规则与法律依据说明”

## 三，图片类复审魔法棒

推荐按钮：

1. 复核图片文字风险
2. 复核图片内容风险
3. 定位高风险区域
4. 解释判定依据

推荐说明：

- 适用于图片复审场景
- 需要同时覆盖 OCR 文字风险和图片视觉内容风险
- 重点解决“风险区域在哪里”“是文字问题还是图片内容问题”“依据是什么”

## 当前 demo 中的落地方向

当前 demo 已按以下逻辑实现：

- 若用户审核的是文字内容，则显示“文字类复审魔法棒”
- 若用户上传的是文稿文件，则显示“文件类复审魔法棒”
- 若用户上传的是图片，则显示“图片类复审魔法棒”
- 用户在复审弹框内完成对话后，不会立即替换第二屏结果
- 只有点击“更新复审结果”按钮后，第二屏结果才会被统一更新

## 后续 PRD 可继续补充的点

- 不同魔法棒动作对应的系统回答模板
- 不同物料类型下，复审结果如何映射回第二屏卡片
- “误判”“漏判”“边界模糊”三种复审场景的产品定义
- 用户点击“更新复审结果”后的结果替换规则与状态提示
